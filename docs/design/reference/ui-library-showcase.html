Choose your UI foundation

Every Groundwork project picks exactly one UI source of truth before the first screen is built: an existing component library, or a bespoke system designed with the design skill. This page explains the choice; every other page is a live, clickable impression of one option.

1

Bring your brand, or build it here

Have a brand guide (colors, fonts, logo)? The design skill imports it and maps it onto tokens. No brand yet? The same skill interviews you and derives one with you, so you are never blocked on "we have no designer".

2

Pick one foundation

Browse the pages on the left. Every demo uses the same components (buttons, form, tabs, modal, toast) so you compare style, not content. Interact with everything: it behaves as it would in a real app.

3

Record it, then theme it

The choice becomes a decision record. Your brand tokens are applied through the library's own theming layer (or your token file, if bespoke). From then on the agent may only build UI through that foundation.

OptionBest whenTrade-off
Component library (pages on the left) You want speed, tested accessibility, and a coherent look with little design effort. Theme it with your brand colors and fonts. Your app looks like the library's family unless you theme deliberately. You inherit its dependencies and upgrade cycle.
Custom / bespoke (last page) The look is part of the product's value, you have strong brand requirements, or you want zero UI dependencies. More design decisions up front. The design skill carries you through them, but budget real time for it.

How the choice lands in the process

  • When: during the design skill, after the stack is chosen. The stack constrains the menu (most libraries below target React; other stacks have their own equivalents, which the stack skill researches live).
  • What gets recorded: a decision record in docs/decisions/ naming the foundation and why; the token mapping in docs/design/DESIGN.md section 3; the theme file in code as the SSOT.
  • What it enforces: one foundation per project. Mixing libraries, or dropping to raw CSS beside a library, needs a new decision record first.
  • Your brand always wins: the library provides structure and behavior; your colors, type, and radius come from your tokens. A themed shadcn app and a themed Mantine app can both carry your brand.

Honest caveats

  • These demos are hand-built impressions in plain CSS, not the libraries' real code. They show the style register, not pixel truth.
  • Library facts (popularity, maintenance, license) age fast. Before deciding, the stack skill verifies the current state from the official docs; never decide from memory or from this page alone.
