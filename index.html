Groundwork

Eén map met regels die elke AI-assistent van eerste idee naar geleverde, onderhouden software leidt. Op enterprise-niveau.

Hoe het werkt De grenzen
AGENTS.mdhet rulebook dat elke assistent leest
## Session protocol

1. Read the handoff block at the
   top of docs/state/STATE.md
   before anything else.
2. Work one task at a time.
   Propose exactly one next step.
3. Before ending a session:
   update STATE.md.
4. End every message with ⚓ to
   confirm these rules are loaded.

Echt fragment. Kopieer de map, zeg "begin", en dit protocol stuurt elke sessie.

AI is krachtig, maar zonder structuur bouwt het rommel

Een AI-assistent schrijft snel code. Maar hij vergeet wat er gisteren gebeurde, gokt soms op verouderde kennis, dwaalt af naar dingen die niemand vroeg, en kan per ongeluk je gegevens laten rondslingeren. Groundwork lost precies die vier dingen op.

Vijf eenvoudige onderdelen die samenwerken

Je hoeft geen regel code te begrijpen. Groundwork is in de kern een set afspraken en automatische controles die de assistent dwingen netjes en veilig te werken.

Zes redenen, eerlijk uitgelegd

Klik op een reden om te zien wat het betekent en hoe Groundwork het echt waarmaakt. Geen beloftes, alleen ingebouwde werkwijzen.

Skills, elk een expert voor eén taak

Een skill is een uitgeschreven werkwijze die de assistent erbij pakt op het juiste moment. Klik op een kaart voor de uitleg in gewone taal.

Eén doorlopende route

Groundwork begeleidt het hele leven van een product, niet alleen het bouwen. Klik op een fase.

Controles die niets laten passeren

Deze draaien automatisch bij elke opslag en publicatie. Ze kosten geen tokens en kunnen niet stiekem worden uitgezet: een dichte poort is informatie, geen obstakel.

Wat Groundwork niet is

Vertrouwen ontstaat door eerlijkheid. Dit zijn de grenzen, ronduit benoemd.
